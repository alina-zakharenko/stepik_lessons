Отличительные особенности у кода, экспортированного из рекордера, по сравнению с кодом, написанным вручную:
минусы
- нет asserts - если не добавить вручную, то тест всегда будет проходить
- тестовые данные негибкие, то есть, если уже будет email зарегистрирован, то повторная регистрация упадет, то есть опять
необходимо ручное вмешательство
- добавление fixture, создние conftest - вручную

плюсы
+ не надо тратить время на написание локаторов
+ классы и методы создаются поумолчанию
+ для начала есть setup_method и teardown_method
+ по умолчанию задается размер окна браузера

Итог: наверное, возможно использовать кода из рекордера в качестве основы для теста, с последующим расширением.
